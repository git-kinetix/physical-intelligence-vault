Exploration 02

The Embedding Space
& SIGReg

How the Sketch Isotropic Gaussian Regularizer prevents representation collapse and keeps the embedding space informative with a single loss term.

Representation collapse is the central failure mode of Joint Embedding Predictive Architectures. When the encoder learns to map every input to the same embedding vector, prediction becomes trivially easy — but the representation carries no useful information. SIGReg prevents this by enforcing that the distribution of embeddings stays close to an isotropic Gaussian.

Collapse vs. Regularization Playground

The characteristic function of a distribution uniquely identifies it. For a standard Gaussian, φ(t) = exp(−t²/2). SIGReg tests whether the embeddings' empirical characteristic function matches this reference along many random projection directions.

Characteristic Function

The beauty of SIGReg is that it simultaneously enforces variance (embeddings must spread out), isotropy (embeddings must spread evenly in all directions), and decorrelation (different dimensions must be independent) — all through a single scalar loss term.

LSIGReg = ∫ |φemp(t) − e−t²/2|² · e−t²/2 dt    averaged over 1024 random projections

Why Collapse Is Bad

Previous JEPAs needed six separate loss terms to prevent collapse: variance, covariance, contrastive, attention entropy, capacity, and prediction losses. SIGReg replaces all the regularization terms with one, reducing tunable hyperparameters from six to one.

The SIGReg Algorithm

In the actual codebase, the SIGReg forward pass is only 6 lines of code (see module.py:25-36), yet it replaces five separate regularization losses from prior work.
