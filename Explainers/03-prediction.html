Part 03

Prediction Pipeline & AdaLN-zero

The representation is only as good as the task it is trained for. In LeWM, that task is next-state prediction: given a window of 3 past frame embeddings and the actions taken between them, predict where the next frame will land in embedding space.

Predicting in Embedding Space

The prediction happens entirely in the learned embedding space — never in pixel space. This is what makes LeWM a Joint Embedding Predictive Architecture: it predicts embeddings, not images. The MSE between predicted and actual embeddings provides the gradient signal that shapes both the predictor and the encoder.

AdaLN-zero: Action-conditioned Modulation

How does the predictor know what action was taken? Through Adaptive Layer Normalization with Zero-initialization (AdaLN-zero). Rather than concatenating actions to the input, the action embedding modulates the transformer's internal representations by adjusting the normalization statistics — shifting, scaling, and gating the signal at each layer.

At initialization, the linear layer that produces the modulation parameters has all weights and biases set to zero. This means shift=0, scale=0, gate=0, so the block initially behaves as an identity function. Training gradually learns how to modulate the signal based on the action taken.

Sliding Context Window

With history_size=3 and num_preds=1, each training sample is a sequence of 4 frames. The first 3 provide context; the 4th is the prediction target. During evaluation, the predictor chains predictions autoregressively — each predicted embedding becomes part of the context for the next prediction.

Loss Computation & Gradient Flow

The prediction loss is a simple MSE between the predicted and target embeddings. But there is a crucial subtlety: gradients flow back through the predictor AND the encoder. The prediction loss trains both networks simultaneously — the encoder must produce embeddings that are predictable, while the predictor must learn to predict them accurately.
