# AMP: Adversarial Motion Priors for Stylized Physics-Based Character Control

## Summary
AMP introduces adversarial motion priors as a fully automated approach for training physics-based character controllers that produce naturalistic, stylized behaviors without requiring hand-designed imitation objectives or motion selection mechanisms. The core idea is to replace the multi-component tracking-based reward functions used in prior work (e.g., [[DeepMimic]]'s pose/velocity/end-effector/COM matching) with a learned discriminator that distinguishes between motion transitions from a reference dataset and those generated by the policy. This discriminator serves as a style reward that automatically encourages the policy to produce motions indistinguishable from the reference data, while a separate task reward specifies the high-level goal (e.g., reach a target heading, strike an object, navigate obstacles). The adversarial formulation, based on GAIL but extended for state-only demonstrations, enables the system to handle large unstructured motion datasets without requiring motion segmentation, phase labels, or explicit motion selection.

The method is evaluated on a diverse cast of characters (34-DOF humanoid, 59-DOF T-Rex, 64-DOF dog) across challenging motor control tasks including target heading, target location, obstacle traversal, soccer dribbling, striking, and stepping stones. When trained on multi-clip datasets (e.g., 434 seconds of locomotion data across 56 clips from 8 subjects), AMP automatically learns to compose and transition between different motion styles, producing emergent gait transitions (walk-to-run) and skill compositions (run-leap-roll for obstacle traversal) without any explicit motion planning. Single-clip comparisons show that AMP achieves pose errors comparable to tracking-based methods (e.g., 0.030 m vs 0.018 m for walking), while offering dramatically simpler reward engineering and natural handling of multi-clip datasets.

Published at SIGGRAPH 2021 (ACM Transactions on Graphics, Vol. 40, No. 4) by researchers at UC Berkeley and Shanghai Jiao Tong University, AMP represents a pivotal paper in the physics-based character animation lineage, bridging [[DeepMimic]]'s single-clip motion tracking paradigm to [[ASE]]'s large-scale adversarial skill embeddings. The adversarial motion prior concept has become a standard building block in subsequent work on humanoid control, robotic locomotion, and character animation.

## Key Contributions
- Introduces adversarial motion priors that replace hand-designed multi-component imitation rewards with a learned discriminator, eliminating the need for manual reward engineering, phase variables, motion segmentation, or explicit motion selection mechanisms
- Extends GAIL (Generative Adversarial Imitation Learning) to state-only demonstrations, enabling the use of unstructured motion capture datasets that contain only state transitions without corresponding actions
- Demonstrates that composition of disparate skills (gait transitions, run-leap-roll combinations) emerges automatically from the adversarial motion prior when trained on multi-clip datasets, without requiring a high-level motion planner or task-specific annotations
- Achieves motion quality comparable to state-of-the-art tracking-based methods ([[DeepMimic]]) on single-clip imitation while being substantially simpler to implement and extending naturally to multi-clip scenarios
- Evaluates on a diverse cast of characters (humanoid, T-Rex, dog) and challenging tasks (target heading, target location, obstacles, dribbling, striking, stepping stones), demonstrating broad applicability of the adversarial motion prior framework

## Architecture / Method
AMP combines a task-specific reward with a learned adversarial style reward to train control policies that achieve goals while maintaining natural motion quality.

**Adversarial Motion Prior (Discriminator):**

The discriminator D(s, s') is trained on state transitions to distinguish reference motion data from policy-generated transitions. It uses a least-squares GAN formulation minimizing the Pearson chi-squared divergence:

Loss_D = E_{d^M}[(D(Phi(s), Phi(s')) - 1)^2] + E_{d^pi}[(D(Phi(s), Phi(s')) + 1)^2] + (w^gp / 2) * E_{d^M}[||grad_phi D(phi)||^2]

where d^M is the motion dataset distribution, d^pi is the policy distribution, Phi maps states to discriminator features, and w^gp = 10 is the gradient penalty coefficient.

**Discriminator Features (Phi):**
- Root linear and angular velocity (in local frame)
- Local joint rotations (6D continuous rotation encoding)
- Local joint velocities
- 3D end-effector positions (in local frame)

**Style Reward:**

r^S(s_t, s_{t+1}) = max[0, 1 - 0.25 * (D(s_t, s_{t+1}) - 1)^2]

This clipped reward ensures the style signal is bounded in [0, 1] and provides smooth gradients for policy optimization.

**Total Reward:**

r_t = w^G * r^G_t + w^S * r^S_t

where r^G is the task-specific goal reward and r^S is the adversarial style reward. The task reward is specified by a simple function (e.g., velocity matching for heading, distance for location), while the style reward handles all low-level motion quality.

**Policy Architecture:**

- **Policy:** Gaussian policy pi(a_t | s_t, g) = N(mu(s_t, g), Sigma) with two hidden layers of 1024 and 512 ReLU units, followed by a linear output layer; fixed diagonal covariance Sigma
- **Value function:** Separate network with identical architecture
- **Discriminator:** Separate network with similar architecture
- **Training algorithm:** PPO

**Character Morphologies:**

| Character | DOF | Description |
|-----------|:---:|-------------|
| Humanoid | 34 | Standard humanoid character |
| T-Rex | 59 | Dinosaur with tail dynamics |
| Dog | 64 | Quadruped with complex limb structure |

**Training Scale:**
- 100-300 million samples per task
- Approximately 30-140 hours on 16 CPU cores
- Motion data from public mocap libraries (CMU, SFU), custom recordings, and artist-authored keyframe animations

Training on restricted datasets (Walk-only or Run-only) limits the achievable speed range, resulting in lower returns (0.46 and 0.63 respectively) when the target speed falls outside the dataset's coverage.

On single-clip imitation, AMP achieves pose errors within the same order of magnitude as explicit motion tracking ([[DeepMimic]]-style), despite not using any explicit pose matching reward. For most motions, AMP's errors are moderately higher (e.g., Walk: 0.030 vs 0.018 m), which is expected since the discriminator provides an implicit rather than explicit pose objective. Notably, AMP outperforms tracking on Spin (0.047 vs 0.063 m) and Side-Flip (0.124 vs 0.191 m), suggesting the adversarial objective can sometimes discover better strategies than explicit tracking for motions with complex phase structure.

### Ablation Results

Key findings from disabling components:
- **Gradient penalty (w^gp):** The most critical component. Removing it causes large performance fluctuations, training instability, and visual artifacts in the generated motions
- **Velocity features:** Essential for certain motions. Without velocity features in the discriminator, the character learns to hold fixed poses rather than producing dynamic motion (e.g., rolling motion degrades to a static lying pose)

## Metrics Used
- [[Episode Return]] — normalized cumulative reward (0-1 scale) combining task and style objectives; primary metric for task performance evaluation
- [[Motion Naturalness]] — enforced via the adversarial discriminator; the style reward r^S provides a learned, continuous measure of motion quality that replaces hand-designed imitation metrics
- Pose Error — mean per-joint position error (meters) used for single-clip comparison against tracking-based methods; measures geometric fidelity of reproduced motions

## Datasets Used
- [[CMU Motion Capture Database]] — primary source of locomotion, acrobatic, and action motion clips; supplemented with SFU mocap library, custom recordings, and artist-authored keyframe animations
- [[DeepMind Control Suite]] — not directly used, but AMP's tasks (heading, location, obstacles) are comparable benchmarks for continuous control

## Related Papers
- [[DeepMimic]] — direct predecessor by the same lead author; AMP replaces [[DeepMimic]]'s hand-designed multi-component imitation reward with a learned adversarial discriminator, eliminating the need for explicit pose matching, phase variables, and motion selection
- [[ASE]] — direct successor that extends AMP's adversarial motion prior with latent skill embeddings and unsupervised skill discovery, enabling a single pre-trained controller to transfer to multiple downstream tasks via lightweight high-level controllers
- [[CALM]] — extends [[ASE]] (and by lineage, AMP) with a conditional discriminator and motion encoder for directable latent control, enabling users to specify both task goals and motion style
- [[Learning Agile Robotic Locomotion]] — same lead author's sim-to-real quadruped work; uses hand-designed imitation rewards that AMP's adversarial approach aims to replace
- [[Sim-to-Real Transfer]] — same lead author's foundational dynamics randomization work; AMP operates in simulation but its adversarial prior has been subsequently adopted for sim-to-real locomotion pipelines
- [[MVAE]] — VAE-based alternative for character control from motion data; AMP uses adversarial (GAN) training rather than variational inference, which avoids posterior collapse and produces sharper motion quality
- [[MaskedMimic]] — builds on the AMP/[[ASE]] lineage for masked motion completion in physics-based characters
- [[Hierarchical Puppeteer]] — hierarchical visual humanoid control; AMP's adversarial motion prior is conceptually related to the tracker's imitation objective but operates without explicit tracking
- [[Vid2Player3D]] — same lead author's tennis simulation; uses similar physics-based character skills with motion data
- [[DeepMimic]] — predecessor in the [[DeepMimic]] -> AMP -> [[ASE]] -> [[CALM]] lineage of physics-based character animation
