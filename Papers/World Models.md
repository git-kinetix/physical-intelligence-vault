# World Models

## Summary
World Models introduces a framework for training agents in reinforcement learning environments by first learning a compressed generative model of the environment and then training a simple controller policy on top of the learned representations. The key insight is that an agent can build an internal model of its world — learning spatial and temporal abstractions in an unsupervised manner — and then use this model to learn effective behaviors with extremely compact policies (as few as 867 parameters for the controller).

The architecture consists of three components: a Variational Autoencoder (V) that compresses visual observations into a compact latent code, a Mixture Density Network combined with an LSTM (M) that predicts future latent codes given past observations and actions, and a simple linear Controller (C) that maps the combined latent state and RNN hidden state to actions. On CarRacing-v0, the full V+M agent achieves a score of 906 ± 21, substantially exceeding the prior best of 838 ± 11. The paper further demonstrates that agents can be trained entirely within "dreams" — hallucinated trajectories generated by the world model — and transfer the learned policy back to the real environment, as shown on the VizDoom Take Cover task.

This paper is widely regarded as a foundational work in the learned world models paradigm for reinforcement learning, directly inspiring the [[DreamerV1|Dreamer]] series and a broad lineage of model-based RL methods that learn behaviors by imagination in latent spaces.

## Key Contributions
- Proposes a modular V(AE) + M(DN-RNN) + C(ontroller) architecture that cleanly separates perception, prediction, and control
- Demonstrates that a simple linear controller with fewer than 1,000 parameters can solve complex tasks when given access to learned world model features
- Shows that agents can be trained entirely inside hallucinated "dream" environments generated by the learned world model and successfully transfer to real environments
- Achieves state-of-the-art score of 906 ± 21 on CarRacing-v0, well above the prior best of 838 ± 11
- Introduces the temperature parameter for controlling stochasticity during dream training, revealing a tradeoff between dream-world exploitation and real-world transfer
- Proposes an iterative training procedure where the agent progressively explores, refines its world model, and improves its policy

## Architecture / Method
The architecture has three distinct components trained in sequence:

**V Model — Vision (Convolutional VAE):**
A convolutional variational autoencoder compresses each 64×64 RGB observation frame into a compact latent vector z of dimension N_z (32 for CarRacing, 64 for VizDoom). The encoder uses 4 convolutional layers with stride-2 and ReLU activations to produce mean (μ) and log-variance (log σ²) vectors, from which z is sampled via the reparameterization trick. The decoder mirrors the encoder with 4 deconvolutional layers to reconstruct the observation. The VAE is trained on 10,000 random rollouts collected from the environment.

**M Model — Memory (MDN-RNN):**
An LSTM with a Mixture Density Network output layer models the temporal dynamics: P(z_{t+1} | a_t, z_t, h_t), where h_t is the LSTM hidden state and a_t is the action. The MDN uses 5 Gaussian mixture components with factored covariance matrices. The LSTM has 256 hidden units for CarRacing and 512 for VizDoom. The M model is trained for 20 epochs on sequences from the same 10,000 rollouts. During dream training, the M model generates complete hallucinated trajectories by iteratively sampling z_{t+1} from the predicted mixture distribution.

**C Model — Controller (Linear):**
A minimal linear model: a_t = W_c [z_t ; h_t] + b_c, where [z_t ; h_t] is the concatenation of the VAE latent and the RNN hidden state. This yields only 867 parameters for CarRacing and 1,088 for VizDoom. The controller is optimized using Covariance Matrix Adaptation Evolution Strategy (CMA-ES) with a population size of 64, performing 16 rollouts per agent per generation.

**Dream Training:**
For VizDoom, the agent is trained entirely within hallucinated environments generated by the M model. The temperature parameter τ controls the stochasticity of the MDN-RNN's sampling during dream generation. Lower τ produces more deterministic but potentially exploitable dreams; higher τ produces noisier dreams that regularize the policy for real-world transfer. The optimal transfer temperature was found to be τ = 1.15, which produces agents that perform best in the actual environment despite not achieving the highest dream score.

## Results

### Table 1: CarRacing-v0 Performance

| Method | Average Score |
|--------|--------------|
| **Full World Model (V+M)** | **906 ± 21** |
| V model + hidden layer | 788 ± 141 |
| V model only | 632 ± 251 |
| OpenAI Gym Leaderboard best | 838 ± 11 |
| A3C (continuous) | 591 ± 45 |

The full V+M agent achieves 906 ± 21, substantially exceeding the prior OpenAI Gym leaderboard best of 838 ± 11. Ablations show that the M model's hidden state h_t is critical: removing it (V model only) drops performance to 632, while adding a hidden layer to the controller without M yields 788. The compact controller with access to both z_t and h_t achieves the best results.

### Table 2: VizDoom Take Cover — Temperature Ablation (Dream vs. Actual)

| Temperature τ | Virtual (Dream) Score | Actual Score |
|---------------|----------------------|--------------|
| 0.10 | 2086 ± 140 | 193 ± 58 |
| 1.00 | 1145 ± 690 | 868 ± 511 |
| **1.15** | 918 ± 546 | **1092 ± 556** |
| 1.30 | 732 ± 269 | 753 ± 139 |
| Random Policy | — | 210 ± 108 |
| Solve threshold | — | 750 |

At low temperature (τ=0.10), the agent achieves near-perfect scores in the dream but fails catastrophically in reality (193), because it exploits deterministic artifacts in the simplified dream. At τ=1.15, dream scores are moderate but real-world transfer is maximized (1092), well above the solve threshold of 750. This reveals a fundamental tension between dream-world optimization and real-world generalization.

## Metrics Used
- [[Cumulative Reward]] — total episode reward, the primary metric for CarRacing-v0 (max 1000)
- Survival Timesteps — number of timesteps survived in VizDoom Take Cover (max 2100, solve threshold 750)
- Model Parameter Efficiency — number of controller parameters needed to solve tasks, demonstrating the compactness of learned policies
- Dream-to-Real Transfer Gap — difference between virtual (dream) and actual environment performance, quantifying the fidelity of the learned world model

## Datasets Used
- [[CarRacing-v0]] (OpenAI Gym) — a 2D top-down racing environment with randomly generated tracks, continuous steering/acceleration/brake controls, and pixel observations (96×96 RGB, resized to 64×64); 10,000 random rollouts used for unsupervised V and M model training
- [[VizDoom Take Cover]] — a first-person 3D survival task where the agent must dodge fireballs thrown by monsters; rewarded by survival timesteps (max 2100); 10,000 random rollouts used for model training

## Related Papers
- [[DreamerV1]] — direct successor that replaces CMA-ES with actor-critic policy learning in the latent space of a Recurrent State-Space Model (RSSM), achieving stronger results on continuous control tasks
- [[DreamerV2]] — extends DreamerV1 with discrete world model representations and achieves human-level Atari performance
- [[DreamerV3]] — generalizes the Dreamer framework to work across diverse domains with a single configuration
- [[IRIS]] — replaces the RSSM with a discrete autoencoder plus autoregressive transformer for world modeling
- [[DIAMOND]] — uses diffusion models instead of discrete tokens for world modeling, preserving visual fidelity
